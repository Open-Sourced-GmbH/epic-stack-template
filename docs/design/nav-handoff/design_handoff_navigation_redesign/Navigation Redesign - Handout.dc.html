01

Überblick & Ziele

Die Anwendung hatte historisch mehrere, voneinander abweichende Kopfzeilen — eine für die Marketing-Seiten, eine für den eingeloggten Bereich, eine für den Admin. Jede pflegte eigene Abstände, Logos und Zustände. Das Redesign ersetzt sie durch eine einzige Navbar-Komponente, deren Erscheinung deterministisch aus drei Eingaben hervorgeht: der aktuellen Route, der Rolle des Nutzers und der Ansicht (Desktop oder Mobile).

Das Ergebnis ist eine Fläche, die sich überall gleich anfühlt, aber pro Kontext genau die richtige Dichte zeigt: aufgeräumt im Marketing, fokussiert im Login, arbeitsfähig in Konto und Admin.

02

Designprinzipien

A
Eine Navbar für jede Fläche
Statt drei getrennter Kopfzeilen löst genau eine Komponente alle Kontexte auf. Was sichtbar ist, entscheiden Bedingungen — nicht Duplikate.
B
Tokens statt Hex-Werte
Jede Farbe, jeder Abstand und jeder Radius referenziert ein Designtoken. Kein Element kennt seinen rohen Farbwert — nur seine Rolle (brand, muted-foreground, …).
C
Dark Mode als Class-Flip
Das Theme ist eine einzige .dark-Klasse an der Wurzel. Alle *-foreground-, Flächen- und Rahmen-Tokens kippen automatisch.
D
Zugänglich von Grund auf
Fokus-Ringe, ARIA-Labels, Tastaturbedienung und ausreichende Trefferflächen sind Teil der Spezifikation, kein nachträglicher Zusatz.

04

resolveNavbar — Auflösungslogik

Eine einzige Funktion bestimmt die Variante. Sie liest Route und Ansicht und liefert das Layout; die Rolle moduliert anschließend nur, welche Bausteine innerhalb dieser Variante erscheinen (siehe Abschnitt 07).

isFull = route ∈ { account, admin }
isMarketing = route ∈ { marketing, blog }
isMinimal = route = login
// Variante = isMinimal ? Minimal : isMarketing ? Marketing : Full
Route Variante Logo/Wortmarke Produkt-Links Section-Sidebar
loginMinimalWortmarke——
marketing · blogMarketingWortmarkeFunktionen · Über · Docs · Blog—
accountFullWortmarke„Zurück zur Website“Konto
adminFullWortmarke„Zurück zur Website“Manage

Hinweis: In Account & Admin führt der einzige Produkt-Link zurück ins Marketing („Zurück zur Website“). Ein separater „Admin“-Eintrag in der Top-Nav entfällt dort bewusst — der Zugang bleibt über das Avatar-Menü erhalten.

05

Navbar-Varianten

Alle Varianten teilen dieselbe Höhe (60 px), dieselbe Innenraum-Logik (linker Cluster / rechter Cluster) und dieselben Tokens. Sie unterscheiden sich nur im Inhalt.

Minimal Login & Registrierung
Maximale Reduktion: nur Logo und Wortmarke. Keine Produkt-Links, kein Avatar, keine Theme-Steuerung im Weg. Die gesamte Aufmerksamkeit liegt auf der Authentifizierungs-Karte. Die Navbar verzichtet sogar auf den unteren Rahmen, damit Kopf und Karte als eine ruhige Fläche wirken.
Marketing Öffentliche Seiten
Links Wortmarke plus Produkt-Navigation (Funktionen, Über, Docs, Blog). Rechts Akzent- und Theme-Steuerung sowie — je nach Rolle — entweder die primäre CTA „Los geht's“ (Gast) oder die Identität (angemeldet). Einladend, aber nicht überladen.
Full Konto & Admin (App-Shell)
Die Arbeitsumgebung. Links Wortmarke und der Rücksprung „Zurück zur Website“; darunter (Desktop) eine Sektions-Sidebar mit gruppierten Einträgen. Rechts Akzent, Theme und die Identität mit Avatar-Menü. Auf Mobile ersetzt ein Hamburger-Menü die linke Navigation und öffnet den Drawer.
06

Linke & rechte Cluster

Die Navbar ist eine justify-between-Zeile mit zwei Clustern. Links liegt Identität der Anwendung und Navigation, rechts liegt persönliche Steuerung.

Linker Cluster

Hamburger (nur Mobile)Öffnet den Drawer. Nur in den Varianten Marketing & Full sichtbar.
Logo + WortmarkeImmer ein Link zur Startseite (title=„Zur Startseite“). Die Wortmarke kann auf Mobile zugunsten von Platz entfallen.
Produkt-LinksMarketing: Funktionen · Über · Docs · Blog. Full: ein einzelner Rücksprung „Zurück zur Website“. Nur auf Desktop.

Rechter Cluster

Akzent-AuswahlPopover mit fünf Akzentfarben (Teal, Blau, Violett, Amber, Rosé). Setzt --brand live. Nur Desktop.
Theme-UmschalterZyklus hell → dunkel → system. Icon spiegelt den Zustand (Sonne / Mond / Laptop).
Primäre CTA„Anmelden“ (Full, Gast) bzw. „Los geht's“ (Marketing, Gast). Verschwindet, sobald angemeldet.
Identität / AvatarAvatar-Initialen + Name (Desktop). Klick öffnet das Menü: Konto, Admin (nur Admins), Abmelden.
07

Rollen & Sichtbarkeit

Drei Rollen — Gast, User, Admin — steuern, welche Bausteine im rechten Cluster und in der Navigation erscheinen. Die Variante selbst bleibt unberührt.

Baustein Gast User Admin
CTA „Anmelden / Los geht's“✓——
Identität / Avatar-Menü—✓✓
Admin-Eintrag (Menü & Drawer)——✓
Akzent & Theme (Desktop)✓✓✓
Section-Sidebar (Konto/Admin)—✓✓

Beim Abmelden in Konto oder Admin wechselt die Route automatisch zurück ins Marketing — ein Gast soll nie auf einer geschützten Fläche stranden.

08

Responsive: Desktop ↔ Mobile

Desktop
  • Produkt-Links inline im linken Cluster
  • Section-Sidebar als feste Spalte (218 px)
  • Avatar mit Name; Akzent- & Theme-Steuerung sichtbar
  • Popover für Akzent & Avatar-Menü
Mobile
  • Hamburger ersetzt die linke Navigation
  • Sidebar wird zum Off-Canvas-Drawer
  • Avatar ohne Name; Steuerung in den Drawer-Footer
  • Geräte-Rahmen 390 px breit, Trefferflächen ≥ 44 px

Der Wechsel ist keine Skalierung, sondern eine Umverteilung: Steuerungen wandern aus der Kopfzeile in den Drawer-Footer, Inline-Links werden zu Listeneinträgen. Die Identität (Wer bin ich? Welches Theme?) bleibt jederzeit in einem Tap erreichbar.

09

Mobile Drawer

Ein Off-Canvas-Panel (86 % Breite, max. 330 px), das von links einfährt. Es ist in drei Bänder gegliedert: Kopf, scrollbare Navigation, Footer mit Identität und Darstellung.

KopfLogo, Wortmarke und Schließen-Schaltfläche auf opaker Fläche.
NavigationKontextabhängig: in Full „Zurück zur Website“ + Sektions-Einträge; in Marketing „Blog“ (und „Admin“ für Admins).
FooterIdentitätszeile (Avatar, Name, Abmelden) und ein Darstellungs-Streifen mit Akzent- und Theme-Schaltern.
Fix
Das Panel ist jetzt blickdicht: Der Akzent-Schimmer (brand-soft, von Natur aus halbtransparent) liegt über einer opaken Grundfläche (background). Zuvor schien der Seiteninhalt durch und vermischte sich mit dem Menü.

Theming

Zwei unabhängige Achsen: Helligkeit (hell / dunkel / system) und Akzent (fünf Marken-Töne). Beide wirken global über CSS-Variablen an der Shell-Wurzel.

Helligkeit
Class-Flip: .dark an der Wurzel kippt alle Tokens. „System“ folgt prefers-color-scheme.
Akzent
Setzt --brand, --primary, --ring gemeinsam. --brand-soft leitet sich per color-mix ab.

Interaktion & Motion

Bewegung ist punktuell und zweckgebunden: Sie erklärt Herkunft (woher kommt das Panel?) und Hierarchie (was ist neu?), ohne je zu verzögern. Standard-Easing für Einblendungen ist cubic-bezier(.22,.61,.36,1).

Element Bewegung Dauer
Drawer-PanelSlide −100 % → 0 + Fade300 ms
BackdropFade-in180 ms
Routen-InhaltTranslate-Y 9 px + Fade340 ms
Tabellenzeilen (Admin)Stagger, 60 ms Versatz400 ms
Popover / DropdownPop-in, scale .98 + Fade140 ms
Aktiv-ZuständeÜbergang bg / Farbe / Schatten180 ms
A11y
Unter prefers-reduced-motion werden alle Dauern und Verzögerungen auf nahezu null gesetzt — die Funktion bleibt, die Bewegung entfällt.

Barrierefreiheit

›ARIA-Labels auf Hamburger („Menü“) und Avatar-Trigger
›Fokus-Ring über --ring (= Akzent)
›Drawer & Menüs: Fokusfalle, esc schließt (Radix)
›Trefferflächen auf Mobile ≥ 44 px (Drawer-Items 46 px)
›Kontrast über Token-Paare (AA in hell & dunkel)
›Reduzierte Bewegung respektiert (s. Abschnitt 14)

17

Änderungsprotokoll

NAV
Top-Nav: „Blog“ → „Zurück zur Website“
In Konto & Admin führt der Produkt-Link nicht mehr zum Blog, sondern zurück ins Marketing — klarer Rücksprung aus der App-Shell.
NAV
„Admin“ aus der Top-Nav entfernt
In Konto & Admin war der Admin-Link redundant. Der Zugang bleibt über das Avatar-Menü (und im Marketing für Admins) erhalten.
FIX
Drawer blickdicht gemacht
Der Marken-Wash liegt jetzt über einer opaken Grundfläche; der Seiteninhalt scheint nicht mehr durch das offene Menü.
MOT
Motion-Pass
Drawer-Slide, Routen-Einblendung, Zeilen-Stagger und weiche Aktiv-Übergänge ergänzt — alle unter Berücksichtigung reduzierter Bewegung.
